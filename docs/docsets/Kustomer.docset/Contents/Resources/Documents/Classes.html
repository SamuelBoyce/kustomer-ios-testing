Server Trust Evaluators

  • KUSRevocationTrustEvaluator

    An evaluator which Uses the default and revoked server trust evaluations allowing you to control whether to validate the host provided by the challenge as well as specify the revocation flags for testing for revoked certificates. Apple platforms did not start testing for revoked certificates automatically until iOS 10.1, macOS 10.12 and tvOS 10.1 which is demonstrated in our TLS tests. Applications are encouraged to always validate the host in production environments to guarantee the validity of the server’s certificate chain.

    Colors used in the Kustomer UI

    To override colors, create a new color set in your Assets.xcassets folder called KustomerColor.COLOR_NAME where COLOR_NAME is one of our color names.

  • KustomerOptions

    Initialization options. Pass them to KustomerClient.configure(...) override any settings within the sdk.

    If you don’t want to override any settings and use the default ones, you can skip creating this object. You can’t change these after calling KustomerClient.configure(...).

  • Settings

    Settings for your org. Fetched from the REST API. Change them on the Kustomer website, and restart your app to see the new ones. They are saved locally so you can use parts of the Kustomer SDK while offline (assuming you’ve configured and opened the Kustomer UI (regular SDK) or configured and called start(...) once (build your own UI SDK) at least once with an active internet connection).
